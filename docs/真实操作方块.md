# 工业真实机器操作说明

当前工业系统已经支持 `real_machine_recipe` 步骤。这个步骤用于让 NPC 把材料真实放进机器中，等待机器自己产出，而不是通过 `craft_recipe` 凭空生成物品。

## 核心机制

`real_machine_recipe` 不替代旧的 `craft_recipe`。旧配方仍然可以继续使用 `craft_recipe` 做普通工业生产；只有需要真实操作方块实体或机器时，才在步骤中写 `real_machine_recipe`。

执行流程：

1. 从 `point` 找到机器方块位置。
2. 从 `input` 容器检查并取出 `recipe.inputs`。
3. 把输入插入真实机器。
4. 把当前机器输出记录为基线，并把操作状态写入工业箱 `machine_state`。
5. 后续 tick 按 `pollTicks` 轮询机器输出。
6. 当机器输出相对基线达到 `recipe.outputs` 后，步骤完成。
7. 按 `outputPolicy` 决定把产物取到 `output` 容器，还是留在机器里。

## JSON 示例

```json
{
  "type": "real_machine_recipe",
  "point": "furnace",
  "input": "input",
  "output": "output",
  "outputPolicy": "extract_to_output",
  "timeoutTicks": 12000,
  "pollTicks": 20,
  "swing": true
}
```

字段说明：

| 字段 | 说明 |
| --- | --- |
| `point` | 机器方块点位，引用 `points` 中的结构坐标 |
| `input` / `container` | 输入容器，默认 `input` |
| `output` / `container` | 输出容器，默认 `output` |
| `outputPolicy` / `output_policy` | `extract_to_output` 或 `keep_in_machine`，默认 `extract_to_output` |
| `timeoutTicks` / `timeout` | 超时时间，默认 12000 tick |
| `pollTicks` / `poll` | 轮询间隔，默认 20 tick |
| `swing` | 开始插入材料时是否让 NPC 挥手 |

## 输出策略

- `extract_to_output`：机器真实产物出现后，系统从机器输出槽取出并放入 `output` 容器。输出容器满时等待重试，不会删除机器里的产物。
- `keep_in_machine`：机器真实产物留在机器输出槽，工业步骤完成但不搬走物品。

## 配方含义

在 `real_machine_recipe` 中：

- `recipe.inputs` 是要真实插入机器的输入材料。
- `recipe.outputs` 只用于判断机器是否完成，以及限制可以提取哪些产物。
- 步骤里也可以直接写 `inputs` 和 `outputs`。写了就优先使用步骤自己的列表，没写才回退到整条配方。
- `inputs` 支持嵌套 `或` / `与` 逻辑组，真实机器会按实际选中的分支取料并插入机器。
- 工业工人等级倍率不会复制真实机器产物，真实产量以机器本身为准。

## 内置适配器

当前默认支持：

- 可通过 NeoForge `IItemHandler` 或原版 `Container` 访问的通用机器/容器。

默认逻辑不写死熔炉、高炉、烟熏炉等具体方块。原版熔炉这类方块也按通用槽位和通用容器能力处理。复杂模组机器可以后续通过适配器接管：

- 调用 `IndustrialMachineOperationService.registerAdapter(...)` 注册适配器。
- 或监听 `IndustrialMachineOperationEvent.Start`，为本次操作指定适配器。

可监听事件：

- `IndustrialMachineOperationEvent.Start`
- `IndustrialMachineOperationEvent.Tick`
- `IndustrialMachineOperationEvent.Complete`
- `IndustrialMachineOperationEvent.Abort`

## 熔炉通用槽位示例

```json
{
  "id": "iron_ingot_real_furnace",
  "name": "真实熔炉烧铁",
  "heldItem": "minecraft:iron_ore",
  "inputs": [
    { "item": "minecraft:iron_ore", "count": 1 },
    { "item": "minecraft:coal", "count": 1 }
  ],
  "outputs": [
    { "item": "minecraft:iron_ingot", "baseAmount": 1, "ignoreMultiplier": true }
  ],
  "steps": [
    { "type": "move_to_container", "container": "input", "range": 1.2 },
    { "type": "look_at_container", "container": "input" },
    { "type": "inspect_container", "container": "input", "ticks": 20 },
    { "type": "set_held_item", "item": "minecraft:iron_ore" },
    { "type": "move_to", "point": "furnace_stand", "range": 1.2 },
    { "type": "look_at", "point": "furnace" },
    {
      "type": "fill_item",
      "point": "furnace",
      "slot": 0,
      "item": "minecraft:iron_ore",
      "input": "input",
      "targetCount": 64,
      "thresholdCount": 0,
      "swing": true
    },
    {
      "type": "fill_item",
      "point": "furnace",
      "slot": 1,
      "item": "minecraft:coal",
      "input": "input",
      "targetCount": 64,
      "thresholdCount": 0,
      "swing": true
    },
    {
      "type": "real_machine_recipe",
      "point": "furnace",
      "input": "input",
      "output": "output",
      "outputPolicy": "extract_to_output",
      "pollTicks": 20,
      "timeoutTicks": 12000,
      "swing": true,
      "inputs": [],
      "outputs": [
        { "item": "minecraft:iron_ingot", "baseAmount": 1, "ignoreMultiplier": true }
      ]
    }
  ]
}
```

这里的熔炉没有专用代码：`fill_item` 只按 JSON 写的 `slot` 补物品，`real_machine_recipe` 只通过通用容器适配器等待并提取匹配输出。显式写 `"inputs": []` 表示真实机器步骤不再额外塞输入，只等待机器中已有材料烧出产物。

## 中间产物批量加工

如果真实机器一次取出了多个中间产物，后续加工步骤可以使用 `craft_available_recipe`：

```json
{
  "type": "craft_available_recipe",
  "input": "input",
  "output": "output",
  "inputs": [
    { "item": "minecraft:iron_ingot", "count": 1 }
  ],
  "outputs": [
    { "item": "minecraft:iron_nugget", "baseAmount": 9, "ignoreMultiplier": true }
  ]
}
```

它会按输入容器里当前可消耗的数量批量执行配方。例如有 7 个铁锭时，一次消耗 7 个并输出 63 个铁粒。旧的 `craft_recipe` 仍然只执行一轮。

## 状态恢复

真实机器步骤开始后，工业箱会把状态写入 `machine_state`。这可以避免重启、重新进服后重复扣输入材料。

只有以下情况会清理状态：

- 步骤完成。
- 手动暂停。
- 更换配方。
- 解雇工人。
- 控制箱拆除。
- 任务中断。
- 建筑或工业定义变化。

如果机器缺失、输出箱满、机器输入槽不足或超时，步骤会保持等待状态，不会继续推进，也不会凭空产出。  
如果等待真实产物时发现前置 `fill_item` 指定的机器槽低于 `thresholdCount`，流程会清理本次等待状态并回退到补料步骤，避免玩家拿走中间物后一直卡住。
